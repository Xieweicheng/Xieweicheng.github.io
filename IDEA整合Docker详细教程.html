IDEA整合Docker详细教程

发表于 2018-11-15 | 分类于 docker | |
本文介绍IDEA如何整合Docker，以springboot项目为例~

一、简单介绍Docker守护进程（Daemon）

Docker守护进程可以监听三种不同类型的Socket请求：unix，tcp，和fd。

默认情况下，创建unixSocket，/var/run/docker.sock，需要root权限或docker组成员身份。

如果需要远程访问Docker守护进程，则需要启用tcpSocket。注意，默认设置提供对Docker守护进程的未加密和为经过身份验证的直接访问。

  • 未加密（所有人都可以访问，不安全）：
    • dockerd -H tcp:0.0.0.0:2375：使ip地址监听2375所有网络接口上的端口
    • dockerd -H tcp:192.168.59.103:2375：使ip地址监听特定网络接口
  • 加密（只有证书的才能访问，安全）：
    • 通常使用2376与守护进程进行加密通信

以下将分不加密与加密介绍IDEA与Docker的整合。

三、加密

为什么要加密？第二步的配置所有人都可以访问，意味着黑客可以随时黑你的服务器。所以根据官网的说明，可以通过加密通信来保证安全。

加密可参照官网文档一步步操作，传送门

（1）创建保存证书的文件夹

此处我把文件夹创建在root文件夹下

（2）创建CA证书

切换到ac文件夹下，生成CA私钥和公钥

  • 创建私钥，需要输入两次密码
  • 首先输入私钥密码，然后依次输入国家，省，区，组织，邮箱等

（3）创建服务器密钥和证书签名请求（CSR）

  • $Host换成你自己服务器外网的IP或者域名，由于我使用的是虚拟机，这里填写了虚拟机的ip地址192.168.252.128

（4）签署公钥

由于可以通过IP地址和DNS名称建立TLS连接，因此在创建证书时需要指定它们。

建议使用0.0.0.0，即允许所有的ip都访问，但只有拥有证书的人才能够连接成功。此处需要输入一次ca证书密码。

  • 若使用IP则命令如下，多个可用逗号分隔

echo subjectAltName = IP:x.x.x.x,IP:y.y.y.y > extfile.cnf

  • 若使用域名则命令如下

echo subjectAltName = DNS:www.mrxiexie.cn,IP:y.y.y.y > extfile.cnf
